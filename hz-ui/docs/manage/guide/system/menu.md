title: 菜单管理

## 功能概述

菜单管理用于配置系统导航菜单，支持多级菜单、内嵌第三方页面等功能。

::: tip 注意
菜单配置通常由开发人员维护，普通用户只需了解**状态管理**和**第三方页面嵌入**功能即可。
:::

---

## 菜单类型

| 类型 | 说明 | 是否显示在导航 |
|------|------|---------------|
| 目录 | 分组容器，不对应具体页面 | 否 |
| 菜单 | 具体功能页面，可配置权限 | 是 |
| 内嵌 | 以 iframe 方式嵌入第三方页面 | 是 |

---

## 菜单状态管理

### 状态说明

| 状态 | 效果 |
|------|------|
| 启用 | 菜单在导航栏显示并可访问 |
| 禁用 | 菜单隐藏且无法访问 |

### 操作建议

- 临时下线某个功能时，可禁用对应菜单
- 删除菜单前建议先禁用，确认无影响后再删除

---

## 第三方页面嵌入

支持将外部系统以 iframe 方式嵌入平台，实现统一入口。

### 配置步骤

1. 进入菜单管理
2. 新增菜单，设置以下参数：
   - **菜单标识**：`iframe`
   - **路由**：`/iframe`
   - **是否内嵌**：是
   - **访问地址**：第三方系统的实际 URL

### 配置示例

```yaml
菜单名称: Grafana 监控
菜单标识: iframe
路由: /iframe
是否内嵌: YES
访问地址: http://grafana.example.com
```

### 注意事项

- 第三方系统需支持 iframe 嵌入
- 被嵌入系统需配置 CORS 策略
- 建议使用 HTTPS 协议

## 菜单层级结构

### 一级菜单

直接显示在主导航栏，如"首页"、"资产配置"。

### 二级菜单

折叠在一级菜单下，点击展开显示子项。

### 三级及以上

支持更深的菜单层级，建议不超过三级。

---

## 菜单与权限

菜单管理与角色权限联动：

1. 创建菜单时自动生成对应权限标识
2. 在角色管理中分配菜单权限
3. 用户登录后根据角色动态加载菜单

---

## 其他配置

| 配置项 | 说明 |
|--------|------|
| 菜单图标 | 显示在导航栏的图标 |
| 排序号 | 控制菜单显示顺序 |
| 是否缓存 | 页面是否保持缓存状态 |
| 详情页 | 是否带详情页（用于查看/编辑详情） |
