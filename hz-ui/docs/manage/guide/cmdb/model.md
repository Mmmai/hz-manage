## 概述

模型管理是 CMDB 的核心功能，基于模型驱动的资产管理理念，支持用户自定义模型、字段、类型等。

### 模型概念

模型（Model）是资产的抽象定义，描述了一类资产的共同属性。例如：

- **服务器模型**：包含 IP、主机名、操作系统、CPU、内存等字段
- **数据库模型**：包含端口、版本、字符集等字段

::: tip 模型与实例的关系
模型类似于面向对象编程中的"类"，实例则是类的具体"对象"。
:::

## 操作说明

### 创建模型

1. 进入 **模型管理** 页面
2. 点击 **新增** 按钮
3. 填写模型基本信息：
   - **名称**：模型唯一标识，英文，如 `server`
   - **显示名称**：模型展示名称，中文，如 `服务器`
   - **模型分组**：选择所属的模型分组
   - **图标**：选择模型图标（可选）
   - **描述**：模型说明（可选）
4. 点击 **保存**

#### 命名规范

| 项目 | 规范 | 示例 |
|------|------|------|
| 名称 | 小写英文，单词用下划线连接 | `server`, `mysql_database` |
| 显示名称 | 简洁的中文描述 | `服务器`, `MySQL 数据库` |

### 编辑模型

1. 在模型列表中找到目标模型
2. 点击 **编辑** 按钮
3. 修改模型信息
4. 点击 **保存**

::: warning 内置模型限制
内置模型（built_in=true）不可删除，但可以编辑和添加字段。
:::

### 删除模型

1. 在模型列表中找到目标模型
2. 点击 **删除** 按钮
3. 确认删除操作

::: danger 删除模型的影响
删除模型会同时删除：
- 该模型的所有字段定义
- 该模型的所有实例数据
- 该模型相关的关联关系
- 该模型相关的唯一约束

请谨慎操作！
:::

## 实例名称模板

实例名称模板用于自动生成实例的名称，支持动态拼接字段值。

### 配置位置

在模型编辑页面的 **实例名称模板** 部分进行配置。

### 模板语法

模板是一个 JSON 数组，每个元素可以是一个固定字符串或字段引用：

```json
[
  {"field": "hostname"},
  "-",
  {"field": "ip"}
]
```

### 配置示例

#### 示例 1：服务器实例名称

```
字段：hostname = "web-server", ip = "192.168.1.100"
模板：[{"field": "hostname"}, "-", {"field": "ip"}]
结果：web-server-192.168.1.100
```

#### 示例 2：数据库实例名称

```
字段：db_name = "user_db", port = "3306"
模板：[{"field": "db_name"}, ":", {"field": "port"}]
结果：user_db:3306
```

#### 示例 3：固定前缀

```
字段：env = "prod", app = "api"
模板：["srv-", {"field": "env"}, "-", {"field": "app"}]
结果：srv-prod-api
```

### 最佳实践

1. **保持唯一性**：确保生成的名称在模型内唯一
2. **包含关键信息**：将用于识别的字段加入模板
3. **控制长度**：避免生成过长的名称
4. **使用分隔符**：使用 `-`、`:` 等分隔符提高可读性

## 字段管理

### 创建字段

1. 进入模型详情页
2. 切换到 **字段** 标签
3. 点击 **新增字段**
4. 填写字段信息：
   - **名称**：字段英文标识
   - **显示名称**：字段中文展示名
   - **字段类型**：选择数据类型
   - **是否必填**：设置必填选项
   - **默认值**：设置默认值（可选）
   - **验证规则**：选择验证规则（可选）
   - **字段分组**：选择所属分组（可选）
   - **描述**：字段说明（可选）
5. 点击 **保存**

### 字段类型

| 类型 | 说明 | 适用场景 | 示例 |
|------|------|----------|------|
| string | 单行文本 | 名称、标识、短文本 | 主机名、IP 地址 |
| text | 多行文本 | 长文本内容 | 描述、备注 |
| boolean | 布尔值 | 是/否判断 | 是否启用、是否高可用 |
| enum | 枚举选择 | 固定选项 | 操作系统类型、环境 |
| json | JSON 数据 | 结构化数据 | 标签、配置信息 |
| integer | 整数 | 数量、编号 | CPU 核数、端口号 |
| float | 浮点数 | 小数数值 | 内存大小、磁盘容量 |
| password | 密码 | 敏感信息 | 数据库密码、SSH 密码 |
| date | 日期 | 日期信息 | 购买日期、到期日期 |
| datetime | 日期时间 | 时间戳信息 | 创建时间、更新时间 |
| model_ref | 模型引用 | 关联其他模型 | 所属机房、关联应用 |

### 字段属性

| 属性 | 说明 |
|------|------|
| 必填 | 该字段在创建实例时必须填写 |
| 可编辑 | 该字段在实例创建后是否可以修改 |
| 内置 | 系统预置字段，不可删除 |
| 默认值 | 新建实例时自动填入的值 |
| 单位 | 数值字段的单位（如 GB、MB） |

## 字段组管理

字段组用于在实例详情页中将相关字段组织在一起展示。

### 创建字段组

1. 进入模型详情页
2. 切换到 **字段组** 标签
3. 点击 **新增字段组**
4. 填写字段组信息：
   - **名称**：字段组标识
   - **显示名称**：字段组展示名称
   - **描述**：字段组说明（可选）
5. 点击 **保存**

### 将字段添加到分组

1. 进入 **字段** 标签
2. 编辑目标字段
3. 在 **字段分组** 下拉框中选择目标分组
4. 点击 **保存**

### 内置字段组

系统会自动创建一个默认字段组（如"基本信息"），用于容纳未分组的字段。

## 业务场景示例

### 场景一：创建服务器模型

1. **基本信息**
   - 名称：`server`
   - 显示名称：`服务器`
   - 模型分组：`base_model`
   - 图标：`Server`

2. **字段配置**
   | 字段名 | 显示名 | 类型 | 必填 | 验证规则 |
   |--------|--------|------|------|----------|
   | hostname | 主机名 | string | 是 | 正则：`^[a-z0-9-]+$` |
   | ip | IP 地址 | string | 是 | IPv4 验证 |
   | cpu | CPU 核数 | integer | 是 | 范围：1-128 |
   | memory | 内存 | integer | 是 | 范围：1-1024 |
   | os | 操作系统 | enum | 是 | 枚举：CentOS, Ubuntu, Windows |
   | datacenter | 所属机房 | model_ref | 是 | 引用机房模型 |

3. **实例名称模板**
   ```json
   [{"field": "hostname"}, "-", {"field": "ip"}]
   ```
   结果示例：`web-server-192.168.1.100`

4. **字段分组**
   - 基本信息：hostname, ip
   - 硬件配置：cpu, memory
   - 系统信息：os
   - 所属资源：datacenter

### 场景二：创建数据库模型

1. **基本信息**
   - 名称：`database`
   - 显示名称：`数据库`
   - 模型分组：`app_model`
   - 图标：`Database`

2. **字段配置**
   | 字段名 | 显示名 | 类型 | 必填 | 验证规则 |
   |--------|--------|------|------|----------|
   | db_name | 数据库名 | string | 是 | 正则：`^[a-zA-Z0-9_]+$` |
   | port | 端口 | integer | 是 | 范围：1024-65535 |
   | version | 版本 | string | 否 | - |
   | charset | 字符集 | enum | 否 | 枚举：utf8, utf8mb4, gbk |
   | server | 所属服务器 | model_ref | 是 | 引用服务器模型 |

3. **实例名称模板**
   ```json
   [{"field": "db_name"}, "-", {"field": "port"}]
   ```
   结果示例：`user_db-3306`

## 最佳实践

### 模型设计原则

1. **单一职责**：一个模型只描述一类资产
2. **合理抽象**：避免创建过于通用的"万能模型"
3. **适度细化**：字段数量建议控制在 20 个以内
4. **考虑扩展**：预留扩展字段或使用 JSON 字段存储灵活数据

### 字段设计原则

1. **命名规范**
   - 使用英文小写字母和下划线
   - 名称应具有自描述性
   - 避免使用数据库关键字

2. **类型选择**
   - IP 地址使用 `string` 类型（便于验证）
   - 数量使用 `integer` 类型
   - 敏感信息使用 `password` 类型
   - 关联使用 `model_ref` 类型

3. **验证规则**
   - 关键字段必须设置验证规则
   - IP、端口等使用内置验证
   - 自定义格式使用正则表达式

### 命名规范

| 资源 | 好的命名 | 不好的命名 |
|------|----------|------------|
| 模型名称 | `server`, `mysql_database` | `Server`, `mysql` |
| 字段名称 | `hostname`, `cpu_cores` | `name`, `cpu` |
| 显示名称 | `服务器`, `CPU 核数` | `主机`, `核数` |

## 常见问题

### Q: 模型删除后还能恢复吗？

A: 不能。模型删除会同时删除所有相关数据（字段、实例、关系、约束）。删除前请确认，或考虑先导出数据备份。

### Q: 如何修改已有字段的类型？

A: 字段类型创建后不建议修改。如果必须修改：
1. 确保现有数据兼容新类型
2. 可能需要数据迁移
3. 建议创建新字段，迁移数据后删除旧字段

### Q: 实例名称模板可以后续修改吗？

A: 可以。修改模板后：
- 已创建的实例名称不会自动更新
- 新创建的实例使用新模板
- 可以通过"更新实例名称"功能批量更新

### Q: 一个模型可以有多少个字段？

A: 理论上没有限制，但建议控制在 20-30 个以内：
- 字段过多影响界面展示
- 建议使用字段分组组织
- 考虑拆分为多个关联模型

### Q: 如何处理模型间的循环引用？

A: 模型间允许循环引用，但需要注意：
- 创建实例时分先后顺序
- 或者先创建不引用任何实体的实例
- 再通过编辑添加关联关系
