## 概述

实例分组（Instance Group）是对 CMDB 实例进行逻辑分类和组织的树形结构，支持多级嵌套和多对多关联。

### 分组特点

- **树形结构**：支持多级嵌套，形成层次化组织
- **多对多关系**：一个实例可以属于多个分组
- **空闲池机制**：每个模型都有内置的"空闲池"分组
- **路径索引**：每个分组有完整的路径标识

## 核心概念

### 空闲池（Unassigned Pool）

每个模型都有一个内置的"空闲池"分组：

- **分组名**：`未分组` 或 `空闲池`
- **内置标识**：built_in=true
- **作用**：存放未分配到业务分组的实例
- **可删除**：否

::: tip 空闲池的作用
1. 实例创建后默认进入空闲池
2. 删除实例前必须先移入空闲池
3. 作为实例的临时存放地
:::

### 树形结构

实例分组采用树形结构，支持父子关系：

```
生产环境
├── Web 应用
│   ├── 前端服务器
│   └── 后端服务器
├── 数据服务
│   ├── MySQL 集群
│   └── Redis 集群
└── 中间件
    └── 消息队列

测试环境
├── 测试 Web
└── 测试数据库
```

### 分组路径

每个分组都有完整的路径标识：

- **顶级分组**：`生产环境`
- **子分组**：`生产环境/Web 应用`
- **更深层级**：`生产环境/Web 应用/前端服务器`

::: tip 路径的作用
路径用于：
1. 快速定位分组
2. 筛选和查询实例
3. 权限控制
:::

## 操作说明

### 创建分组

1. 进入 **实例分组** 页面
2. 选择目标模型
3. 点击 **新增分组**
4. 填写分组信息：
   - **分组名称**：分组标识，如 `web_servers`
   - **显示名称**：分组展示名称，如 `Web 服务器`
   - **父分组**：选择上级分组（可选，不选则为顶级分组）
   - **排序**：分组显示顺序（可选）
5. 点击 **保存**

::: tip 分组命名规范
- 分组名称：英文小写，如 `production`, `web_servers`
- 显示名称：中文，简洁明了，如 `生产环境`, `Web 服务器`
- 避免使用特殊字符
:::

### 编辑分组

1. 在分组树中找到目标分组
2. 点击 **编辑** 按钮
3. 修改分组信息
4. 点击 **保存**

::: warning 路径变更影响
修改分组的父节点会改变路径：
- 影响分组的路径索引
- 可能影响基于路径的查询
- 系统会自动更新所有子分组的路径
:::

### 删除分组

1. 在分组树中找到目标分组
2. 点击 **删除** 按钮
3. 确认删除操作

::: danger 删除分组的限制
1. 内置分组（如空闲池）不可删除
2. 删除分组前，需要先移除分组下的所有实例
3. 删除分组会同时删除所有子分组
:::

## 实例管理

### 添加实例到分组

#### 方式一：从实例列表添加

1. 进入 **资源实例** 页面
2. 勾选目标实例
3. 点击 **设置分组**
4. 选择目标分组
5. 点击 **保存**

#### 方式二：从分组页面添加

1. 进入 **实例分组** 页面
2. 选择目标分组
3. 点击 **添加实例**
4. 从实例列表中选择实例
5. 点击 **保存**

### 从分组中移除实例

1. 进入 **实例分组** 页面
2. 选择目标分组
3. 在实例列表中勾选要移除的实例
4. 点击 **移除实例**
5. 确认移除操作

::: tip 移除实例的影响
移除实例不会删除实例：
- 实例仍然存在
- 可以添加到其他分组
- 如果不属于任何分组，自动进入空闲池
:::

### 拖拽操作（如果界面支持）

1. 在分组树中按住实例
2. 拖拽到目标分组
3. 释放鼠标

::: tip 拖拽限制
- 拖拽只能在同一模型内进行
- 跨模型拖拽无效
- 拖拽到不同分组会建立新的关联关系
:::

## 分组类型

### 内置分组

系统自动创建的分组：

| 分组 | 说明 | 可删除 |
|------|------|--------|
| 未分组/空闲池 | 存放未分配实例 | 否 |

### 自定义分组

用户创建的分组：

- **按环境划分**：生产环境、测试环境、开发环境
- **按业务划分**：电商系统、支付系统、用户中心
- **按层级划分**：一级分组、二级分组、三级分组
- **按功能划分**：Web 服务、数据服务、中间件

## 业务场景示例

### 场景一：按环境分组

```
生产环境
├── Web 集群
├── 数据库集群
└── 缓存集群

测试环境
├── 测试 Web
└── 测试数据库

开发环境
├── 开发 Web
└── 开发数据库
```

**用途**：
- 区分不同环境的实例
- 按环境部署应用
- 环境级别的权限控制

### 场景二：按业务线分组

```
电商平台
├── 用户中心
│   ├── 用户服务
│   └── 认证服务
├── 订单中心
│   ├── 订单服务
│   └── 支付服务
└── 商品中心
    ├── 商品服务
    └── 搜索服务

运维平台
├── 监控系统
└── 日志系统
```

**用途**：
- 按业务线组织资源
- 业务级别的成本核算
- 业务级别的权限控制

### 场景三：按应用层级分组

```
电商网站
├── 接入层
│   ├── Nginx 网关
│   └── CDN 节点
├── 应用层
│   ├── Web 服务
│   └── API 服务
├── 服务层
│   ├── 用户服务
│   ├── 订单服务
│   └── 支付服务
└── 数据层
    ├── MySQL 主库
    ├── MySQL 从库
    └── Redis 集群
```

**用途**：
- 清晰展示应用架构
- 便于故障定位和排查
- 支持拓扑可视化

## 分组权限

### 权限控制

实例分组支持权限控制：

- **查看权限**：控制用户能否查看分组内的实例
- **编辑权限**：控制用户能否编辑分组内的实例
- **删除权限**：控制用户能否删除分组内的实例
- **管理权限**：控制用户能否管理分组结构

::: tip 权限继承
子分组可以继承父分组的权限配置。
:::

### 权限配置建议

1. **按团队授权**
   - 开发团队：开发环境分组
   - 运维团队：生产环境分组
   - 测试团队：测试环境分组

2. **按业务授权**
   - 业务负责人：对应业务分组
   - 系统管理员：所有分组

3. **按角色授权**
   - 只读用户：查看权限
   - 运维人员：编辑权限
   - 管理员：管理权限

## 最佳实践

### 分组设计原则

1. **层次清晰**
   - 不超过 4 层
   - 每层含义明确
   - 避免过深嵌套

2. **粒度适中**
   - 分组不宜过细（管理成本高）
   - 分组不宜过粗（失去分类意义）
   - 建议每个分组 10-50 个实例

3. **稳定优先**
   - 分组结构设计要考虑未来扩展
   - 避免频繁调整分组结构
   - 预留扩展空间

### 分组命名规范

**环境分组**：
- 英文：`production`, `testing`, `development`
- 中文：`生产环境`, `测试环境`, `开发环境`

**业务分组**：
- 英文：`ecommerce`, `payment`, `user_center`
- 中文：`电商平台`, `支付系统`, `用户中心`

**功能分组**：
- 英文：`web_cluster`, `db_cluster`, `cache_cluster`
- 中文：`Web 集群`, `数据库集群`, `缓存集群`

### 分组管理建议

1. **定期清理**
   - 清理空分组（无实例的分组）
   - 合并相似分组
   - 删除过时的分组

2. **文档记录**
   - 记录分组结构设计
   - 记录分组命名规范
   - 记录分组管理流程

3. **权限审计**
   - 定期审查分组权限
   - 移除不必要的权限
   - 更新权限变更记录

## 常见问题

### Q: 一个实例可以属于多个分组吗？

A: 可以。实例与分组是多对多关系：
- 一个实例可以同时属于多个分组
- 例如：一台服务器可以同时在"生产环境"和"Web 集群"分组中

### Q: 删除分组后，分组下的实例会怎样？

A: 删除分组：
1. 不会删除实例
2. 实例与该分组的关联关系被删除
3. 实例可能仍在其他分组中
4. 如果不属于任何分组，自动进入空闲池

### Q: 如何移动分组？

A: 移动分组（改变父节点）：
1. 编辑分组
2. 修改"父分组"字段
3. 保存更改
4. 系统会自动更新分组路径

::: warning 路径变更影响
移动分组会改变分组的路径，可能影响基于路径的查询和权限配置。
:::

### Q: 空闲池可以删除吗？

A: 不可以。空闲池是内置分组（built_in=true）：
- 系统自动创建
- 不可删除
- 不可重命名
- 用于存放未分配实例

### Q: 如何批量移动实例到其他分组？

A:
1. 进入 **资源实例** 页面
2. 筛选或勾选目标实例
3. 点击 **设置分组**
4. 选择目标分组
5. 保存更改

或使用拖拽功能（如果界面支持）。

### Q: 分组路径有什么作用？

A: 分组路径用于：
1. 快速定位分组（如 `生产环境/Web 应用/前端服务器`）
2. 筛选和查询实例（如查询某路径下的所有实例）
3. 权限控制（如按路径授权）
4. 拓扑可视化（如展示分组的层次结构）

### Q: 如何限制实例只能属于一个分组？

A: 系统默认支持多对多关系。如果需要限制：
- 在业务层面控制（如培训用户、制定流程）
- 或在应用层面添加验证逻辑

### Q: 分组的实例数量有限制吗？

A: 理论上没有限制，但建议：
- 每个分组 10-50 个实例（便于管理）
- 如果实例过多，考虑创建子分组
- 或使用标签、筛选等功能辅助管理
