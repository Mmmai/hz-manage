## 概述

资源实例管理是 CMDB 的数据维护页面，用于创建、编辑、删除资产实例。

### 实例概念

实例（Instance）是模型的具体数据对象，类似于面向对象编程中的"对象"。

- **实例名称**：根据模板自动生成或手动录入
- **实例数据**：存储在字段中，每个字段对应一个值
- **实例分组**：可以属于多个分组（多对多关系）
- **输入模式**：手动录入、表格导入、自动发现

## 操作说明

### 创建实例

1. 进入 **资源实例** 页面
2. 选择目标模型
3. 点击 **新增** 按钮
4. 填写实例信息：
   - **实例名称**：自动生成（使用模板）或手动输入
   - **字段值**：根据模型定义填写各字段值
   - **必填字段**：标 * 号的字段必须填写
   - **引用字段**：从下拉列表中选择关联实例
5. 点击 **保存**

::: tip 实例名称生成
如果配置了实例名称模板，系统会自动根据字段值生成实例名称。
也可以手动输入实例名称（需关闭模板功能）。
:::

### 编辑实例

1. 在实例列表中找到目标实例
2. 点击 **编辑** 按钮
3. 修改字段值
4. 点击 **保存**

::: warning 字段可编辑性
某些字段可能被设置为"不可编辑"（editable=false），这些字段在创建后无法修改。
:::

### 删除实例

1. 在实例列表中找到目标实例
2. 点击 **删除** 按钮
3. 确认删除操作

::: danger 删除限制
实例删除有如下限制：
1. 实例必须先移入"空闲池"分组
2. 如果实例有关联关系，需要先删除关系
3. 删除操作不可逆，请谨慎操作
:::

### 批量操作

#### 批量删除

1. 勾选要删除的实例
2. 点击 **批量删除** 按钮
3. 确认删除操作

::: warning 批量删除限制
批量删除需要：
1. 所有实例都已在"空闲池"中
2. 所有实例都没有关联关系
:::

#### 批量导出

1. 筛选或勾选要导出的实例
2. 点击 **导出** 按钮
3. 选择导出范围：
   - **当前页**：仅导出当前页的实例
   - **所有结果**：导出所有筛选结果
4. 下载 Excel 文件

## 实例状态

### 输入模式（input_mode）

实例创建时记录的输入方式：

| 模式 | 说明 | 场景 |
|------|------|------|
| manual | 手动录入 | 用户手动创建实例 |
| import | 表格导入 | 通过 Excel 批量导入 |
| discover | 自动发现 | 自动发现工具创建 |

### 使用模板（using_template）

指示实例名称是否使用了模板：

- `true`：使用实例名称模板生成
- `false`：手动输入实例名称

## 导入导出

### 导入功能

#### 导入步骤

1. 点击 **导入** 按钮
2. 下载导入模板
3. 填写数据到模板
4. 上传填写好的 Excel 文件
5. 系统验证数据
6. 确认导入

#### 导入格式要求

- **第一行**：字段名称（中文显示名）
- **数据行**：按照字段类型填写数据
- **必填字段**：必须填写
- **引用字段**：确保目标实例存在

#### 导入验证

导入时会进行以下验证：
- 字段类型验证
- 必填字段验证
- 验证规则验证
- 唯一约束验证
- 引用关系验证

::: tip 导入错误处理
如果导入失败，系统会返回详细的错误信息，包括：
- 错误行号
- 错误字段
- 错误原因
修正后可重新导入。
:::

### 导出功能

#### 导出选项

| 选项 | 说明 |
|------|------|
| 当前页 | 仅导出当前页的实例 |
| 所有结果 | 导出所有筛选结果 |

#### 导出内容

导出的 Excel 文件包含：
- 所有字段的值
- 字段显示名作为表头
- 密码字段不包含明文（脱敏）

::: warning 密码字段导出
密码字段在导出时不会包含明文值，这是安全机制。
如需备份密码数据，请使用其他方式（如数据库备份）。
:::

## 字段显示偏好

用户可以自定义字段的显示和隐藏。

### 设置字段偏好

1. 进入实例列表
2. 点击 **列设置** 或 **字段偏好** 按钮
3. 勾选或取消勾选字段
4. 点击 **保存**

### 重置偏好

1. 点击 **重置偏好** 按钮
2. 确认重置操作

::: tip 偏好与权限
字段偏好只影响显示，不影响权限。即使隐藏字段，有权限的用户仍可在详情页查看。
:::

详细配置请参考 [字段偏好](./field-preference.md)。

## 业务场景示例

### 场景一：批量导入服务器

1. **准备模板**
   - 下载服务器模型的导入模板
   - 模板包含：主机名、IP、CPU、内存、操作系统、所属机房

2. **填写数据**

   | 主机名 | IP | CPU | 内存 | 操作系统 | 所属机房 |
   |--------|-----|-----|------|----------|----------|
   | web-server-01 | 192.168.1.100 | 8 | 16 | CentOS 7 | 北京机房 |
   | web-server-02 | 192.168.1.101 | 8 | 16 | CentOS 7 | 北京机房 |
   | db-server-01 | 192.168.1.200 | 16 | 32 | CentOS 7 | 北京机房 |

3. **导入数据**
   - 上传填写好的 Excel
   - 系统验证数据格式
   - 确认导入

### 场景二：更新服务器配置

1. **筛选服务器**
   - 模型：服务器
   - 操作系统：CentOS 7

2. **批量编辑**
   - 勾选目标服务器
   - 点击 **批量编辑**
   - 选择要更新的字段
   - 批量设置新值

3. **保存更改**

::: tip 批量编辑
批量编辑可用于：
- 批量更新字段值
- 批量设置实例分组
- 批量添加标签
:::

### 场景三：下线服务器

1. **移除关联关系**
   - 删除服务器与应用的关联关系
   - 删除服务器与机房的关联关系（如需要）

2. **移入空闲池**
   - 编辑实例
   - 设置实例分组为"空闲池"
   - 保存

3. **删除实例**
   - 点击 **删除** 按钮
   - 确认删除

## 最佳实践

### 实例命名规范

- 使用有意义的名称
- 包含关键识别信息
- 保持命名一致性

| 好的命名 | 不好的命名 |
|----------|------------|
| `web-server-prod-01` | `server1` |
| `mysql-db-master` | `db` |
| `nginx-gateway-01` | `nginx` |

### 数据质量管理

1. **必填字段**
   - 关键字段设置为必填
   - 确保数据完整性

2. **验证规则**
   - 为关键字段配置验证规则
   - 防止错误数据进入

3. **唯一约束**
   - 为唯一字段设置约束
   - 防止重复数据

4. **定期清理**
   - 下线实例及时移入空闲池
   - 定期清理空闲池中的过期实例

### 批量操作建议

1. **导入前验证**
   - 检查数据格式
   - 确认引用实例存在
   - 小批量测试导入

2. **导出后备份**
   - 导出数据作为备份
   - 定期导出关键数据
   - 异地存储备份文件

3. **批量编辑谨慎**
   - 批量编辑前先筛选
   - 确认筛选结果正确
   - 备份原始数据

## 常见问题

### Q: 为什么无法删除实例？

A: 实例删除有以下限制：
1. 实例不在"空闲池"分组中
2. 实例存在关联关系

解决方法：
1. 先将实例移入"空闲池"
2. 删除所有关联关系

### Q: 导入失败怎么办？

A: 导入失败通常由以下原因导致：
1. 数据格式不符合字段类型
2. 必填字段未填写
3. 引用字段的目标实例不存在
4. 违反唯一约束

查看错误信息，修正数据后重新导入。

### Q: 导出的 Excel 中为什么没有密码？

A: 这是安全机制：
- 密码字段使用国密算法加密
- 导出时不包含明文密码
- 如需备份密码，使用数据库备份

### Q: 实例名称可以修改吗？

A: 可以修改，但需要注意：
- 如果使用了实例名称模板，修改相关字段后名称会自动更新
- 如果手动输入名称，可以直接编辑
- 实例名称在模型内必须唯一

### Q: 如何批量移动实例到其他分组？

A:
1. 勾选目标实例
2. 点击 **批量编辑**
3. 设置实例分组
4. 保存更改

或使用拖拽功能（如果界面支持）。

### Q: 字段偏好设置对所有用户生效吗？

A: 不。字段偏好是用户个人的设置：
- 每个用户可以设置自己的偏好
- 不影响其他用户的界面
- 登录后自动加载个人偏好

### Q: 如何恢复误删除的实例？

A:
1. 检查 **资产审计** 日志
2. 查看删除操作记录
3. 根据审计记录中的字段值重新创建
4. 或联系系统管理员从数据库备份恢复

::: tip 审计日志
所有删除操作都会记录在审计日志中，包含删除前的完整数据。
:::
