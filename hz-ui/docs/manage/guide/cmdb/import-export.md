## 概述

CMDB 提供了强大的数据导入导出功能，支持批量数据操作，提高数据管理效率。

### 功能价值

- **批量导入**：快速初始化大量资产数据
- **数据迁移**：从其他系统迁移数据到 CMDB
- **数据备份**：定期导出资产数据作为备份
- **数据交换**：与其他系统进行数据交换

## 导入功能

### 导入流程

#### 1. 下载导入模板

1. 进入 **资源实例** 页面
2. 选择目标模型
3. 点击 **导入** 按钮
4. 点击 **下载模板**
5. 保存 Excel 模板文件

::: tip 模板内容
导入模板包含：
- 第一行：字段显示名（中文）
- 第二行及以后：示例数据（可选）
- 所有字段的类型和格式要求
:::

#### 2. 填写数据

使用 Excel 或其他表格工具填写数据：

**基本要求**：
- 第一行必须是字段显示名（与模板一致）
- 数据需符合字段类型约束
- 必填字段不能为空
- 日期时间格式需符合系统要求

**示例数据**：

| 主机名 | IP 地址 | CPU 核数 | 内存 (GB) | 操作系统 | 所属机房 |
|--------|---------|----------|-----------|----------|----------|
| web-server-01 | 192.168.1.100 | 8 | 16 | CentOS 7 | 北京机房 |
| web-server-02 | 192.168.1.101 | 8 | 16 | CentOS 7 | 北京机房 |
| db-server-01 | 192.168.1.200 | 16 | 32 | CentOS 7 | 北京机房 |

::: warning 数据格式注意事项
- IP 地址：使用标准 IPv4 格式
- 日期：使用 `YYYY-MM-DD` 格式
- 日期时间：使用 `YYYY-MM-DD HH:MM:SS` 格式
- 枚举字段：值必须与枚举选项一致
- 引用字段：使用实例名称（确保目标实例存在）
:::

#### 3. 上传文件

1. 在导入页面点击 **上传文件**
2. 选择填写好的 Excel 文件
3. 系统自动解析文件

#### 4. 数据验证

系统会自动验证数据：

**验证内容**：
- 字段类型验证
- 必填字段验证
- 验证规则验证（正则、范围、长度等）
- 唯一约束验证
- 引用关系验证

**验证结果**：
- ✅ 验证通过：数据可以导入
- ❌ 验证失败：显示错误信息

#### 5. 确认导入

1. 查看验证结果
2. 如果有错误，修正后重新上传
3. 如果验证通过，点击 **确认导入**
4. 等待导入完成

::: tip 导入进度
大数据量导入可能需要较长时间：
- 系统会显示导入进度
- 导入过程中不要关闭页面
- 导入完成后会显示结果统计
:::

### 导入错误处理

#### 常见错误类型

| 错误类型 | 说明 | 解决方法 |
|----------|------|----------|
| 格式错误 | 数据格式不符合字段类型 | 检查数据格式，修正后重新导入 |
| 必填字段为空 | 必填字段未填写 | 填写必填字段 |
| 验证失败 | 不符合验证规则 | 修正数据使其符合规则 |
| 唯一约束冲突 | 违反唯一约束 | 修改数据使其唯一 |
| 引用实例不存在 | 引用字段的目标实例不存在 | 先创建目标实例，或修改引用字段 |
| 重复数据 | 实例名称或唯一字段重复 | 修改数据使其唯一 |

#### 错误信息示例

```
第 3 行，字段"IP 地址"：格式不正确，应为有效的 IPv4 地址
第 5 行，字段"所属机房"：引用的实例"北京机房"不存在
第 7 行，字段"主机名"：与现有实例重复
```

::: tip 错误处理建议
1. 下载错误报告
2. 在 Excel 中修正错误数据
3. 重新上传文件
4. 确认无错误后再导入
:::

### 密码字段导入

**导入方式**：
- 密码字段在导入时支持明文
- 系统会自动使用国密算法加密存储
- 导入后显示时自动脱敏

**安全建议**：
- 导入完成后删除包含明文密码的 Excel 文件
- 不要将包含密码的模板发送给无关人员
- 使用安全的文件传输方式

## 导出功能

### 导出类型

#### 当前页导出

导出当前页的实例数据。

1. 筛选或查询目标实例
2. 在当前页查看数据
3. 点击 **导出** 按钮
4. 选择 **当前页**
5. 下载 Excel 文件

**适用场景**：
- 导出少量数据
- 导出当前筛选结果
- 快速获取数据副本

#### 所有结果导出

导出所有查询结果的实例数据（不分页）。

1. 筛选或查询目标实例
2. 点击 **导出** 按钮
3. 选择 **所有结果**
4. 下载 Excel 文件

**适用场景**：
- 导出大量数据
- 导出完整筛选结果
- 数据备份

### 导出内容

导出的 Excel 文件包含：

- **所有字段**：实例的所有字段值
- **字段显示名**：表头使用字段显示名
- **实例名称**：第一列通常是实例名称
- **创建时间**：包含实例创建时间
- **更新时间**：包含实例更新时间

::: warning 密码字段不导出
密码字段在导出时不包含明文值：
- 这是安全机制
- 导出的 Excel 中密码列为空或显示 `******`
- 如需备份密码，使用数据库备份
:::

### 导出格式

导出的 Excel 文件特点：

- **文件格式**：`.xlsx`（Excel 2007+）
- **编码**：UTF-8
- **日期格式**：`YYYY-MM-DD` 或 `YYYY-MM-DD HH:MM:SS`
- **数值格式**：保留原始格式
- **布尔值**：`true` / `false`

## 大数据量处理

### 大数据量导入

#### 分批导入

如果数据量很大（超过 1000 行），建议分批导入：

1. **拆分文件**
   - 将大文件拆分为多个小文件
   - 每个文件 500-1000 行
   - 逐个文件导入

2. **使用异步导入**（如果支持）
   - 上传文件后异步处理
   - 在任务列表查看进度
   - 完成后查看结果

#### 导入性能优化

1. **减少验证规则**
   - 临时放宽验证规则
   - 导入后再恢复规则

2. **关闭索引**（管理员操作）
   - 导入前暂时关闭索引
   - 导入完成后重建索引

3. **使用事务**（自动）
   - 系统自动使用事务
   - 失败时自动回滚

### 大数据量导出

#### 分批导出

如果数据量很大，建议分批导出：

1. **筛选缩小范围**
   - 使用模型筛选
   - 使用时间范围筛选
   - 使用其他条件筛选

2. **分页导出**
   - 分页导出每页数据
   - 手动合并多个文件

3. **使用异步导出**（如果支持）
   - 提交导出任务
   - 在任务列表查看进度
   - 完成后下载文件

#### 导出性能优化

1. **限制导出字段**
   - 只导出必要的字段
   - 减少数据量

2. **使用筛选条件**
   - 缩小导出范围
   - 减少导出数据量

3. **选择合适的导出方式**
   - 小数据量：使用当前页导出
   - 大数据量：使用所有结果导出

## 最佳实践

### 导入最佳实践

1. **准备阶段**
   - 下载最新模板
   - 检查字段定义和验证规则
   - 确认引用实例存在

2. **填写数据**
   - 使用 Excel 填写数据
   - 遵循数据格式要求
   - 填写所有必填字段

3. **验证数据**
   - 在 Excel 中进行基本检查
   - 检查必填字段是否填写
   - 检查数据格式是否正确

4. **分批导入**
   - 大数据量分批导入
   - 先导入少量数据测试
   - 确认无误后再批量导入

5. **验证结果**
   - 导入后检查数据
   - 确认数据完整性
   - 检查关联关系

### 导出最佳实践

1. **定期备份**
   - 定期导出重要数据
   - 保存多个版本备份
   - 异地存储备份文件

2. **筛选导出**
   - 使用筛选条件缩小范围
   - 只导出需要的数据
   - 减少文件大小

3. **文件管理**
   - 使用有意义的文件名
   - 包含导出日期和模型
   - 示例：`server_20240321.xlsx`

4. **安全注意**
   - 导出文件包含敏感信息
   - 妥善保管导出文件
   - 及时删除不需要的导出文件

### 数据安全

1. **文件安全**
   - 导入文件包含敏感信息
   - 使用加密文件传输
   - 导入后删除源文件

2. **密码安全**
   - 不要在导入文件中包含明文密码
   - 或导入后立即修改密码
   - 删除包含密码的 Excel 文件

3. **访问控制**
   - 导入导出需要相应权限
   - 审计导入导出操作
   - 记录操作日志

## 常见问题

### Q: 导入失败后，已导入的数据会回滚吗？

A: 是的。导入使用事务机制：
- 导入失败时自动回滚
- 不会保留部分数据
- 确保数据一致性

### Q: 导入时如何处理重复数据？

A:
- **更新模式**：如果实例存在则更新，不存在则创建
- **跳过模式**：跳过重复数据，只导入新数据
- **报错模式**：遇到重复数据时报错，停止导入

具体模式取决于系统配置。

### Q: 导出的 Excel 可以直接用于导入吗？

A: 可以，但需要注意：
1. 导出的 Excel 可以作为导入模板
2. 第一行的字段显示名不能修改
3. 密码字段导出为空，需要重新填写
4. 修改数据后可以重新导入

### Q: 导入时引用字段如何填写？

A: 引用字段填写目标实例的实例名称：
- 例如：`所属机房` 字段填写 `北京机房`
- 系统会查找名称为 `北京机房` 的机房实例
- 如果不存在，导入会失败

### Q: 如何导入大量数据？

A:
1. 拆分为多个小文件（每个 500-1000 行）
2. 逐个文件导入
3. 或使用异步导入功能（如果支持）
4. 导入前验证数据，减少失败率

### Q: 导出时为什么没有密码字段？

A: 这是安全机制：
- 密码字段使用国密算法加密
- 导出时不包含明文密码
- 防止密码泄露
- 如需备份密码，使用数据库备份

### Q: 导入后实例名称是如何生成的？

A:
- 如果配置了实例名称模板，系统会根据字段值自动生成
- 如果没有配置模板，需要手动填写实例名称
- 实例名称在模型内必须唯一

### Q: 如何处理导入时的字符编码问题？

A:
- 确保导入文件使用 UTF-8 编码
- 或将 Excel 文件另存为 `.xlsx` 格式
- 避免使用 `.csv` 格式（可能存在编码问题）
