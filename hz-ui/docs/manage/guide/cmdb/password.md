## 概述

CMDB 中的密码字段使用国密算法加密存储，提供安全的密码管理能力。

### 密码字段特点

- **加密存储**：使用国密算法加密，不存储明文
- **显示脱敏**：界面显示时自动脱敏为 `******`
- **权限控制**：查看/显示密码需要特定权限
- **审计记录**：所有密码操作都被审计

## 密码加密机制

### 国密算法

CMDB 使用国家密码管理局认可的加密算法：

- **算法类型**：SM4（国密对称加密算法）
- **密钥管理**：系统统一管理加密密钥
- **加密强度**：符合国家密码标准

::: tip 国密算法优势
- 国家认证的安全算法
- 加密强度高
- 符合合规要求
:::

### 加密流程

1. **用户输入密码**（明文）
2. **前端传输**：使用 HTTPS 加密传输
3. **后端加密**：使用国密算法加密
4. **存储密文**：将加密后的密文存储到数据库

### 解密流程

1. **读取密文**：从数据库读取加密后的密码
2. **验证权限**：检查用户是否有查看密码权限
3. **解密明文**：使用国密算法解密
4. **显示明文**：在界面显示明文密码

## 密码字段配置

### 创建密码字段

1. 进入 **模型管理** 页面
2. 选择目标模型
3. 切换到 **字段** 标签
4. 新增字段：
   - **字段名称**：如 `db_password`
   - **显示名称**：如 `数据库密码`
   - **字段类型**：选择 `password`
   - **是否必填**：根据需求设置
5. 点击 **保存**

### 密码字段属性

| 属性 | 说明 |
|------|------|
| 字段类型 | password |
| 存储方式 | 国密算法加密 |
| 显示方式 | 脱敏显示 `******` |
| 验证规则 | 密码规则验证 |
| 默认值 | 不支持设置默认值 |

::: warning 密码字段默认值
密码字段不支持设置默认值：
- 每个实例需要单独设置密码
- 或通过导入时填写明文密码
- 系统自动加密存储
:::

## 显示密码

### 显示密码权限

查看或显示密码需要特定权限：

- **显示密码权限**：控制用户是否可以查看密码明文
- **权限配置**：在权限管理中配置
- **审计记录**：所有显示密码操作都会被审计

### 显示密码操作

1. 进入 **资源实例** 页面
2. 找到包含密码字段的实例
3. 点击 **显示密码** 按钮（如果有权限）
4. 系统验证权限
5. 显示明文密码

::: tip 显示密码限制
- 需要有"显示密码"权限
- 操作会被审计记录
- 密码显示一段时间后自动隐藏
:::

### 密码复制

1. 显示密码后，点击 **复制** 按钮
2. 密码被复制到剪贴板
3. 粘贴到需要的地方

::: warning 密码复制安全
- 密码复制后应尽快使用
- 使用后清空剪贴板
- 不要通过不安全渠道传输密码
:::

## 密码重新加密

### 重新加密场景

以下情况需要重新加密密码：

1. **加密密钥更新**：系统更新加密密钥
2. **算法升级**：升级加密算法
3. **安全审计**：定期重新加密确保安全

### 重新加密操作（管理员）

1. 进入 **系统维护** 页面
2. 找到 **密码重新加密** 功能
3. 点击 **执行** 按钮
4. 系统重新加密所有密码字段
5. 等待完成

::: tip 重新加密建议
- 定期重新加密（如每年一次）
- 在业务低峰期执行
- 执行前备份密码数据
:::

## 密码重置

### 重置密码

1. 进入 **资源实例** 页面
2. 找到目标实例
3. 点击 **编辑**
4. 修改密码字段
5. 点击 **保存**

**操作影响**：
- 旧密码被新密码替换
- 新密码被加密存储
- 操作被审计记录

### 批量重置密码

1. 勾选目标实例
2. 点击 **批量编辑**
3. 选择密码字段
4. 设置新密码（或生成随机密码）
5. 点击 **保存**

::: warning 批量重置风险
批量重置会影响多个实例：
- 确认选中实例正确
- 重置后需要通知相关人员
- 或使用随机密码并单独分发
:::

## 导入密码

### 导入时填写密码

1. 下载导入模板
2. 在密码字段列填写明文密码
3. 上传文件
4. 系统自动加密存储

::: tip 导入密码安全建议
- 使用临时密码
- 导入后立即修改
- 删除包含明文密码的 Excel 文件
- 使用安全的文件传输方式
:::

### 导入时密码为空

如果导入时密码字段为空：
- 实例创建成功
- 密码字段为空
- 后续可以编辑填写密码

## 安全建议

### 密码管理最佳实践

1. **使用强密码**
   - 包含大小写字母、数字、特殊字符
   - 长度至少 12 位
   - 避免使用常见密码

2. **定期更换密码**
   - 关键密码定期更换
   - 如数据库密码、SSH 密码
   - 建议每季度更换一次

3. **不同系统不同密码**
   - 避免使用相同密码
   - 降低密码泄露风险
   - 便于密码管理

4. **限制密码访问**
   - 只给必要人员分配"显示密码"权限
   - 定期审计密码访问记录
   - 及时撤销离职人员权限

5. **安全传输密码**
   - 不要通过邮件、聊天工具传输密码
   - 使用密码管理工具
   - 或使用临时密码 + 强制修改

### 密码审计

1. **定期审计**
   - 检查密码访问记录
   - 发现异常访问
   - 及时处理安全风险

2. **密码泄露处理**
   - 立即重置泄露密码
   - 检查其他密码是否安全
   - 加强访问控制

3. **合规要求**
   - 遵守行业密码管理规范
   - 满足合规审计要求
   - 定期进行安全评估

## 常见问题

### Q: 密码字段可以使用默认值吗？

A: 不可以。密码字段：
- 不支持设置默认值
- 每个实例需要单独设置
- 防止密码泄露

### Q: 导出时密码会包含明文吗？

A: 不会。导出时：
- 密码字段不包含明文
- 显示为 `******` 或空
- 这是安全机制
- 如需备份密码，使用数据库备份

### Q: 如何查看密码明文？

A:
1. 需要"显示密码"权限
2. 点击"显示密码"按钮
3. 系统解密后显示
4. 操作会被审计记录

### Q: 密码字段可以用于哪些场景？

A: 密码字段适用于：
- 数据库密码
- SSH 登录密码
- API 密钥
- 应用凭证
- 其他敏感凭证

### Q: 密码加密后可以解密吗？

A: 可以。密码加密：
- 使用对称加密算法
- 可以解密还原明文
- 但需要"显示密码"权限
- 所有解密操作被审计

### Q: 如何确保密码安全？

A: 建议措施：
1. 使用强密码
2. 定期更换密码
3. 限制密码访问权限
4. 审计密码访问记录
5. 使用安全的传输方式
6. 及时处理密码泄露

### Q: 密码字段和普通字段有什么区别？

A:
- **密码字段**：加密存储、显示脱敏、需要权限查看
- **普通字段**：明文存储、直接显示、无特殊权限

对于敏感信息，使用密码字段更安全。
