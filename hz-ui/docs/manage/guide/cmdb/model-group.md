## 概述

模型分组（Model Group）是对 CMDB 模型进行逻辑分类的组织方式，帮助用户更好地管理和查找模型。

### 功能特点

- **层级组织**：将相关模型归类到同一分组下
- **内置分组**：系统预置的基础分组
- **自定义分组**：用户可根据业务需求创建
- **编辑控制**：支持控制分组的可编辑性

## 内置分组

系统预置了以下模型分组：

| 分组名称 | 显示名称 | 说明 | 是否可删除 |
|----------|----------|------|------------|
| base_model | 基�础模型 | 系统基础模型，如服务器、网络设备等 | 否 |
| app_model | 应用模型 | 应用相关模型，如应用、数据库等 | 否 |
| resource_model | 资源模型 | 资源相关模型，如机房、机柜等 | 否 |

::: warning 内置分组限制
内置分组（built_in=true）不可删除，确保系统基础模型的稳定性。
:::

## 操作说明

### 创建分组

1. 进入 **模型管理** 页面
2. 点击 **模型分组** 标签
3. 点击 **新增** 按钮
4. 填写分组信息：
   - **名称**：英文标识，如 `monitoring_model`
   - **显示名称**：中文展示名，如 `监控模型`
   - **描述**：分组用途说明
5. 点击 **保存**

### 编辑分组

1. 在模型分组列表中找到目标分组
2. 点击 **编辑** 按钮
3. 修改分组信息
4. 点击 **保存**

::: tip 分组名称规范
- 使用英文小写字母和下划线
- 采用 `category_model` 格式
- 避免使用特殊字符和空格
:::

### 删除分组

1. 在模型分组列表中找到目标分组
2. 点击 **删除** 按钮
3. 确认删除操作

::: danger 删除限制
- 内置分组（built_in=true）不可删除
- 不可编辑的分组（editable=false）不可删除
- 删除分组前，请确保该分组下没有关联的模型
:::

## 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 分组唯一标识，英文 |
| verbose_name | string | 是 | 分组显示名称，中文 |
| description | text | 否 | 分组描述信息 |
| built_in | boolean | 否 | 是否内置，默认 false |
| editable | boolean | 否 | 是否可编辑，默认 true |

## 最佳实践

### 分组设计原则

1. **按业务领域划分**
   ```
   - 基础设施：机房、机柜、网络设备
   - 计算资源：服务器、虚拟机、容器
   - 应用组件：应用、数据库、中间件
   - 监控告警：监控项、告警规则
   ```

2. **按技术栈划分**
   ```
   - Web 技术栈：Nginx、Tomcat、Node.js
   - 数据技术栈：MySQL、Redis、MongoDB
   - 消息队列：Kafka、RabbitMQ
   ```

3. **按环境划分**
   ```
   - 生产环境模型
   - 测试环境模型
   - 开发环境模型
   ```

### 分组命名规范

- 使用简洁的英文名称
- 采用统一的后缀，如 `_model`
- 避免使用拼音缩写

| 好的命名 | 不好的命名 |
|----------|------------|
| `infrastructure_model` | `jichu` |
| `database_model` | `db` |
| `monitoring_model` | `jk_model` |

## 常见问题

### Q: 为什么有些分组无法删除？

A: 内置分组（built_in=true）或不可编辑分组（editable=false）受系统保护，无法删除。这是为了确保系统基础模型的稳定性。

### Q: 模型分组删除后，分组下的模型会怎样？

A: 模型不会随分组删除而删除，只是失去了分组关联。模型会出现在"未分组"列表中。

### Q: 如何重新组织现有模型的分组？

A:
1. 进入模型管理
2. 选择要移动的模型
3. 编辑模型
4. 修改"模型分组"字段
5. 保存更改

### Q: 一个模型可以属于多个分组吗？

A: 不可以。一个模型只能属于一个模型分组，但可以通过其他方式（如标签、前缀）进行分类。

### Q: 模型分组和实例分组有什么区别？

A:
- **模型分组**：对"类"进行分组，即对模型定义分类
- **实例分组**：对"对象"进行分组，即对具体实例分类

例如：
- 模型分组：将"服务器模型"、"网络设备模型"归为"基础模型"组
- 实例分组：将"web-server-01"、"db-server-01"归为"生产环境"组
