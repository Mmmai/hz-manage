## 功能概述

资产审计（Audit）记录 CMDB 数据的完整变更历史，支持追溯、审计和数据恢复。

### 审计的价值

- **故障追溯**：快速定位配置变更的时间点和责任人
- **合规审计**：满足企业合规和审计要求
- **数据恢复**：参考历史值恢复误删或错误修改的数据
- **操作分析**：分析用户操作行为和数据变更趋势

## 审计内容

### 记录的操作类型

系统记录以下操作：

| 操作类型 | 说明 | 示例 |
|----------|------|------|
| 创建 | 创建新实例 | 创建服务器实例 |
| 更新 | 修改实例字段值 | 修改服务器 IP 地址 |
| 删除 | 删除实例 | 删除服务器实例 |
| 批量创建 | 批量导入创建实例 | Excel 导入服务器 |
| 批量更新 | 批量修改实例 | 批量更新服务器配置 |
| 批量删除 | 批量删除实例 | 批量删除过期服务器 |

### 审计字段

审计记录包含以下字段：

| 字段 | 说明 |
|------|------|
| 操作时间 | 操作发生的时间 |
| 操作用户 | 执行操作的用户名 |
| 操作类型 | 创建/更新/删除等 |
| 目标模型 | 操作涉及的模型 |
| 目标实例 | 操作涉及的实例（如适用） |
| 变更字段 | 发生变更的字段 |
| 变更前值 | 字段的原始值 |
| 变更后值 | 字段的新值 |
| IP 地址 | 操作用户的 IP 地址 |
| 操作结果 | 成功/失败 |

### 敏感字段处理

敏感字段的审计处理：

| 字段类型 | 处理方式 |
|----------|----------|
| 密码字段 | 不记录原始值，记录 `******` 或 `已加密` |
| 密钥字段 | 不记录原始值，记录 `******` 或 `已加密` |
| 其他敏感字段 | 根据配置决定是否脱敏 |

::: tip 敏感字段配置
敏感字段的审计脱敏可在系统配置中设置。
:::

## 查询审计

### 基础查询

1. 进入 **资产审计** 页面
2. 设置查询条件：
   - **时间范围**：选择日期范围
   - **模型**：筛选特定模型的操作
   - **操作类型**：筛选操作类型
   - **操作用户**：筛选特定用户
3. 点击 **查询**
4. 查看审计结果

### 高级查询

#### 按字段查询

查询特定字段的变更记录：

```
字段名：IP 地址
时间范围：最近 7 天
```

#### 按值查询

查询特定值的变更记录：

```
变更前值：192.168.1.100
变更后值：192.168.1.200
```

#### 按实例查询

查询特定实例的所有变更：

```
实例名称：web-server-01
```

#### 组合查询

组合多个查询条件：

```
时间范围：最近 7 天
模型：服务器
操作用户：admin
操作类型：更新
```

### 查看审计详情

1. 在审计列表中点击记录
2. 查看完整的审计信息：
   - 操作详情
   - 变更前后对比
   - 操作上下文
   - 关联信息

::: tip 审计详情
审计详情包含完整的变更前后值，可用于数据恢复。
:::

## 业务场景示例

### 场景一：故障追溯

**场景**：生产环境服务器故障，需要查询最近的配置变更。

**操作步骤**：
1. 进入 **资产审计** 页面
2. 设置查询条件：
   - 目标实例：故障服务器
   - 时间范围：故障前 24 小时
   - 操作类型：更新
3. 查看变更记录
4. 定位可能导致故障的配置变更
5. 根据变更时间确定责任人

### 场景二：合规审计

**场景**：审计部门要求查看特定用户的操作记录。

**操作步骤**：
1. 进入 **资产审计** 页面
2. 设置查询条件：
   - 操作用户：目标用户
   - 时间范围：审计周期（如一个月）
3. 导出审计记录
4. 提交给审计部门

::: tip 合规审计
合规审计通常需要：
1. 完整的操作记录
2. 不可篡改的日志
3. 长期保存（通常 6 个月以上）
:::

### 场景三：数据恢复

**场景**：实例被误删除，需要恢复数据。

**操作步骤**：
1. 进入 **资产审计** 页面
2. 查询删除操作记录
3. 查看删除前的完整数据（变更前值）
4. 根据审计记录重新创建实例
5. 填写字段值

::: tip 数据恢复建议
- 优先使用数据库备份恢复
- 审计记录用于辅助恢复
- 记录恢复操作以便追溯

### 场景四：操作溯源

**场景**：发现某个字段的值被错误修改，需要查找责任人。

**操作步骤**：
1. 进入 **资产审计** 页面
2. 设置查询条件：
   - 目标实例：问题实例
   - 变更字段：问题字段
   - 操作类型：更新
3. 查看变更记录
4. 确认操作用户和操作时间
5. 联系相关人员确认

## 审计数据管理

### 审计数据保留策略

系统支持以下保留策略：

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 永久保留 | 审计数据永久保存 | 合规要求高的企业 |
| 定期归档 | 超过一定时间的数据归档 | 需要长期保存但减少查询压力 |
| 定期清理 | 超过一定时间的数据删除 | 审计要求较低的场景 |

::: warning 审计数据删除
审计数据删除后不可恢复，请谨慎操作。
建议先归档再清理。
:::

### 审计数据导出

1. 设置查询条件
2. 点击 **导出** 按钮
3. 选择导出格式：
   - **Excel**：便于查看和分析
   - **JSON**：便于数据处理和存储
   - **CSV**：便于导入其他系统
4. 下载文件

### 审计数据安全

**访问控制**：
- 审计数据需要特定权限才能查看
- 敏感审计记录需要更高权限
- 所有审计查询会被记录

**数据加密**：
- 审计数据传输加密
- 审计数据存储加密
- 导出文件加密（可选）

**完整性保护**：
- 审计记录不可修改
- 审计记录不可删除（除管理员清理）
- 审计数据签名验证

## 最佳实践

### 审计查询技巧

1. **使用时间范围**
   - 缩小查询范围
   - 提高查询速度
   - 便于定位问题

2. **组合查询条件**
   - 多条件组合精确查询
   - 避免查询结果过多
   - 使用实例名称精确查找

3. **导出重要数据**
   - 导出审计记录作为备份
   - 定期导出合规审计数据
   - 异地存储导出文件

1. **合规要求**
   - 了解行业合规要求
   - 确定保留期限
   - 实施归档策略

2. **存储优化**
   - 定期归档旧数据
   - 减少在线数据量
   - 提高查询性能

3. **备份策略**
   - 定期备份审计数据

   - 验证备份完整性

1. **定期分析**
   - 分析操作频率
   - 发现异常操作
   - 优化业务流程

2. **安全审计**
   - 监控敏感操作
   - 发现权限滥用
   - 追踪安全事件

## 常见问题

### Q: 审计记录可以修改吗？

A: 不可以。审计记录设计为不可修改：
- 防止篡改历史
- 确保审计可信
- 满足合规要求

### Q: 审计数据会占用很多空间吗？

A: 会。审计数据随时间增长：
- 建议定期归档旧数据
- 或设置数据保留策略
- 清理超过保留期的数据

### Q: 如何查询所有删除操作？

A:
1. 进入 **资产审计** 页面
2. 操作类型选择 **删除**
3. 设置时间范围
4. 查询结果

### Q: 审计记录包含密码字段吗？

A: 包含，但已脱敏：
- 密码字段记录为 `******` 或 `已加密`
- 不记录明文密码
- 确保密码安全

### Q: 审计数据可以用于法律证据吗？

A: 可以，但需要注意：
- 确保审计系统安全
- 防止审计数据篡改
- 建立审计管理流程
- 咨询法律顾问

### Q: 如何区分批量操作和单个操作？

A:
- 操作类型：批量操作会标记为 `批量创建`、`批量更新`、`批量删除`
- 审计记录：批量操作会汇总记录，或每个实例单独记录
- 具体取决于系统实现

### Q: 审计数据可以恢复实例吗？

A: 不能直接恢复，但可以：
1. 查看实例删除前的完整数据
2. 根据审计记录手动重建实例
3. 或使用数据库备份恢复
