## 功能概述

资源检索提供快速查找 CMDB 资产的能力，支持多条件组合查询、正则表达式、关联查询等高级功能。

### 查询价值

- **快速定位**：从大量资产中快速找到目标
- **数据筛选**：按条件筛选出符合条件的资产
- **数据分析**：通过查询分析资产分布
- **批量操作**：对查询结果进行批量处理

## 查询语法

### 基础查询

#### 等于查询

查询字段值等于指定值的实例。

```
字段名: 值
```

**示例**：
```
ip: 192.168.1.100
hostname: web-server-01
os_type: CentOS 7
```

#### 不等于查询

查询字段值不等于指定值的实例。

```
字段名 != 值
```

**示例**：
```
os_type != Windows
status != offline
```

### 高级查询

#### 包含查询

查询字段值包含指定字符串的实例。

```
字段名 ~ 值
```

**示例**：
```
hostname ~ web
description ~ 生产
```

::: tip 包含查询
包含查询不区分大小写（取决于数据库配置）。
:::

#### 正则表达式查询

使用正则表达式匹配字段值。

```
字段名 =~ 正则表达式
```

**示例**：
```
hostname =~ ^web
ip =~ ^192\.168\.
```

#### 多值查询（OR 查询）

查询字段值等于多个值中任意一个的实例。

```
字段名: 值1, 值2, 值3
```

**示例**：
```
os_type: CentOS 7, CentOS 8, Ubuntu 20.04
status: running, pending
```

#### 范围查询

查询字段值在指定范围内的实例。

```
字段名: 最小值-最大值
```

**示例**：
```
cpu_cores: 8-16
memory: 32-64
port: 8000-9000
```

#### 比较查询

使用比较运算符进行查询。

```
字段名 > 值
字段名 >= 值
字段名 < 值
字段名 <= 值
```

**示例**：
```
cpu_cores >= 16
memory >= 64
create_time > 2024-01-01
```

#### 空值查询

查询字段为空或非空的实例。

```
字段名: null
字段名: !null
```

**示例**：
```
description: null      # 描述为空的实例
description: !null     # 描述不为空的实例
```

#### 关联查询

查询关联到指定实例的实例。

```
关联字段名.关联字段名: 值
```

**示例**：
```
datacenter.name: 北京机房          # 所属机房为北京机房
server.hostname: web-server-01     # 关联的服务器主机名
```

#### 反向查询

查询被哪些实例关联的实例。

```
@关联模型.字段名: 值
```

**示例**：
```
@server.hostname: web-server-01    # 被主机名为 web-server-01 的服务器关联的实例
```

### 组合查询

#### AND 查询

多个条件同时满足（默认逻辑）。

```
条件1 AND 条件2 AND 条件3
```

**示例**：
```
os_type: CentOS 7 AND cpu_cores >= 8 AND memory >= 16
```

#### OR 查询

满足多个条件中的任意一个。

```
条件1 OR 条件2 OR 条件3
```

**示例**：
```
os_type: CentOS 7 OR os_type: Ubuntu 20.04
```

#### 混合查询

AND 和 OR 混合使用。

```
(条件1 OR 条件2) AND 条件3
```

**示例**：
```
(os_type: CentOS 7 OR os_type: Ubuntu 20.04) AND cpu_cores >= 8
```

::: tip 运算符优先级
1. 括号 `()` 优先级最高
2. AND 优先级高于 OR
3. 建议使用括号明确逻辑
:::

## 业务场景示例

### 场景一：查找特定网段的服务器

**需求**：查找 IP 在 `192.168.1.0/24` 网段的所有服务器。

**查询语句**：
```
ip =~ ^192\.168\.1\.
```

或使用范围查询：
```
ip: 192.168.1.1-192.168.1.254
```

### 场景二：查找高配置服务器

**需求**：查找 CPU 核数 >= 16 且内存 >= 64GB 的服务器。

**查询语句**：
```
cpu_cores >= 16 AND memory >= 64
```

### 场景三：查找特定状态的应用

**需求**：查找状态为"运行中"或"启动中"的应用。

**查询语句**：
```
status: running, pending
```

或使用 OR 查询：
```
status: running OR status: pending
```

### 场景四：查找未设置描述的实例

**需求**：查找描述字段为空的实例。

**查询语句**：
```
description: null
```

### 场景五：查找特定机房的服务器

**需求**：查找所属机房为"北京机房"的所有服务器。

**查询语句**：
```
datacenter.name: 北京机房
```

### 场景六：查找特定时间段创建的实例

**需求**：查找 2024 年 1 月之后创建的实例。

**查询语句**：
```
create_time > 2024-01-01
```

### 场景七：查找 Web 服务器

**需求**：查找主机名以 `web-` 开头或包含 `web` 的服务器。

**查询语句**：
```
hostname =~ ^web-
```

或：
```
hostname ~ web
```

### 场景八：排除测试环境

**需求**：查找所有非测试环境的实例。

**查询语句**：
```
environment != test
```

### 场景九：查找特定应用的依赖

**需求**：查找被"电商网站"应用关联的所有数据库实例。

**查询语句**：
```
@app.name: 电商网站
```

### 场景十：复合条件查询

**需求**：查找生产环境、CPU >= 8、操作系统为 CentOS 的服务器。

**查询语句**：
```
environment: production AND cpu_cores >= 8 AND os_type: CentOS
```

## 查询结果操作

### 查看详情

1. 点击实例行
2. 查看完整实例信息
3. 查看实例关联关系

### 导出结果

1. 筛选查询结果
2. 点击 **导出** 按钮
3. 选择导出范围：
   - **当前页**：仅导出当前页的实例
   - **所有结果**：导出所有查询结果
4. 下载 Excel 文件

### 批量操作

1. 勾选目标实例
2. 选择批量操作：
   - **批量编辑**：批量修改字段值
   - **批量删除**：批量删除实例（需在空闲池）
   - **设置分组**：批量设置实例分组
   - **导出**：批量导出选中实例

::: tip 批量操作限制
- 批量删除需要实例在空闲池中
- 批量编辑受字段可编辑性限制
- 操作前请确认选中实例正确
:::

## 查询性能优化

### 查询优化建议

1. **使用索引字段**
   - 优先使用有索引的字段（如实例名称、创建时间）
   - 避免对长文本字段进行正则查询

2. **缩小查询范围**
   - 先使用模型筛选
   - 再使用字段条件
   - 最后使用复杂查询

3. **避免全表扫描**
   - 避免以 `~`（包含）开头的模糊查询
   - 避免复杂的正则表达式
   - 避免多值 OR 查询

4. **使用分页**
   - 使用分页浏览结果
   - 避免一次加载大量数据

### 查询缓存

系统会缓存常用查询：
- 最近执行的查询会保留
- 可以快速重新执行
- 缓存有有效期限制

## 最佳实践

### 查询语句编写

1. **使用有意义的字段名**
   - 优先使用业务字段（如 IP、主机名）
   - 避免使用系统字段（如 ID）

2. **使用精确匹配**
   - 精确查询（`:`）性能优于模糊查询（`~`）
   - 能用精确匹配就不用模糊匹配

3. **使用括号明确逻辑**
   - 复杂查询使用括号
   - 避免依赖默认优先级

### 查询结果管理

1. **使用分页**
   - 大量结果使用分页
   - 避免页面卡顿

2. **导出重要数据**
   - 导出查询结果作为备份
   - 定期导出关键数据

3. **保存常用查询**
   - 保存常用查询语句
   - 使用查询别名（如果支持）

### 查询权限

1. **尊重数据权限**
   - 只查询有权限的实例
   - 不尝试绕过权限限制

2. **查询审计**
   - 所有查询会被记录
   - 敏感查询会触发告警

## 常见问题

### Q: 查询不区分大小写吗？

A: 取决于数据库配置：
- 字符串查询通常不区分大小写
- 正则表达式可以使用 `(?i)` 标志控制
- 建议使用小写查询更通用

### Q: 如何查询包含特殊字符的值？

A: 使用转义字符或引号：
```
description: "包含:冒号的值"
hostname: "包含-连字符的值"
```

### Q: 正则查询性能差怎么办？

A: 正则查询优化：
1. 使用更精确的正则表达式
2. 限制正则表达式范围（如添加前缀）
3. 或使用其他查询方式替代

### Q: 可以跨模型查询吗？

A: 当前版本不支持跨模型查询：
- 查询限定在单个模型内
- 如需跨模型，使用关联查询
- 或分别查询后合并结果

### Q: 查询结果数量有限制吗？

A: 有：
- 单次查询最多返回 1000 条（可配置）
- 超过限制需要使用分页
- 或使用导出功能获取全部结果

### Q: 如何保存查询结果？

A:
1. 使用导出功能下载 Excel
2. 或使用浏览器书签保存查询 URL（如果支持）
3. 或记录查询语句

### Q: 查询历史在哪里查看？

A:
- 查询历史记录在审计日志中
- 可以查看用户的查询操作
- 包括查询条件和结果数量
